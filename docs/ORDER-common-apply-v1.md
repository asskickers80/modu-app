# ORDER-common-apply-v1 — 공통 항목 5축 일괄 적용 (자율 판정 모드)

## 모드 (대표 확정)
항목별 승인 없이 끝까지 진행. 판단이 필요한 지점은 Code가 원칙에 따라 스스로 판정하고,
판정 근거를 결과 보고서에 적는다. 멈추는 경우는 딱 3가지만:
(a) 스키마 변경 SQL 필요, (b) 실 DB 데이터 정정 필요, (c) 원칙끼리 충돌해 판정 불가.

## 대상
5축: 창업준비, 운영중, 임대인, 기업회원, 그냥구경.
임대인은 설계 미확정(7개 미결)이므로 "설계와 무관하게 안전한 항목만" 적용하고
나머지는 [임대인-설계후] 태그로 보고서에 유보 목록화.

## 적용 항목 (누적 기록 전체)
1. 홈 매물/객체 카드 전환 (0건=CTA, 1건+=카드) — 축별 "객체" 판정 포함
2. 헤더 진실의 원천 (선언값 < 실제 등록 객체값)
3. 진행 가이드: 관찰 가능 단계 + 전 단계 탭 진입 + 문의 단계 "문의받기·협의시작" 2칸(첫 문의 일시)
   — 축별 가이드 없으면 신설하지 말고 비적용 판정
4. 업종 2단계 (categories.ts/IndustryPicker) — 업종 입력 있는 모든 화면
5. 같은 기능 컴포넌트 추출·재사용 (UnreadDot, ownership.js, conversation.js) 공유 점검, 복제 제거
6. E2급 상세의 소유자 모드
7. negotiating 뱃지 노출 (창업준비 탐색)
8. AI 라벨 전면 제거·모두 화법 (5축 카피 감사)
9. 권위적 어휘 제거
10. 생성 vs 편집 분리 (소개글류)
11. 미출시 상품·미구현 기능 약속 문구 전수 감사 (프리미엄/멤버십 잔존 포함)
12. 지표류: 실카운트(0이면 0)/정직한 준비중, 더미 금지. 문의 지표는 새문의(강조·펄스·UnreadDot)+전체 서브
13. 방문자 열람 자유 + 행동 게이트(returnTo 의도 보존) — 열람 게이트 잔존 전수 확인
14. 행동 게이트 발 가입 온보딩 생략·역할 유지 — 암묵 기본값(seller 오설정류) 함정 점검
15. 사업자 상태 관찰·폐업 자동 비공개 — 사업자번호 가진 축(운영중·기업회원)
