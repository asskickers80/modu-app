# ORDER-landlord-review-prep-v1 — 임대인(소유주) 축 검토 준비 (돌려보며 잡기)

## 모드: 자율 판정. 멈춤 (a)스키마 SQL (b)실DB 정정 (c)판정 불가만.

## 배경
양도인 축 검토 방식(실기기 시뮬레이션 → 발견 즉시 수정 → 누적)을 임대인 축에 적용.
미결 설계는 선결하지 않고 시뮬레이션 중 부딪히는 순서로 결정.

## 1. 미결 전문 발췌 (최우선)
handover_v16 임대인 절 + PROGRESS 임대인 미결 항목 원문 그대로(해석·요약 금지).

## 2. 임대인 축 현황 조사
- 온보딩: A2 임대인→A3(지역/상태/개수)→A7 실동작. 암묵 기본값 함정(A4 seller 오설정).
- E1p 등록: 임대/매각 분기 구현 상태, 동작 범위.
- E2L 상세: 표시 항목, 소유자 모드, 문의 버튼.
- 홈(A7Landlord): 카드 전수(카드명/출처).
- 데이터 모델: 상가 객체 listings 재사용 vs 별도. status 체계.
- D4: 임대인 앞 문의 도착 가능 상태인가.

## 3. 안전 정비만 선실행 (설계 결정 불요)
- 카피 AI·권위·미구현 grep. 온보딩 뒤로가기·재진입. E2L 비로그인 열람.
- ※ 설계 의존(홈카드·가이드·소유자모드·헤더)은 손대지 말 것.

## 4. docs/SIM-LANDLORD-v1.md
신규 이메일 가입→임대인 온보딩→홈→E1p(임대 1회·매각 1회)→E2L→(가능시)타계정 문의→수신.
단계별 확인 + "여기서 미결 질문 N번 걸림" 표시.
