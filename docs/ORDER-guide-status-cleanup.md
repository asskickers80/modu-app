# ORDER: 협의중 status + 진행 가이드 재구성 + 데이터 정리 (묶음)

## A. "협의중(negotiating)" status 도입
1. SQL 제시 후 멈춤 (status 값 추가 + 제약 갱신, 실행은 대표 콘솔) → 대표 확인 후 계속
2. 전환 규칙: published ↔ negotiating ↔ completed. hidden은 현행 유지. completed 후 전환 불가 (현행 규칙 유지)
3. 탐색 노출 정책 (확정): 협의중 매물은 탐색에 계속 노출 + "협의 중" 뱃지, 문의 가능 (레퍼런스: 당근마켓 예약중 — 협의 결렬 대비 대기 수요 유지)
4. E2 소유자 모드 상태 전환에 협의중 추가, 홈 카드 뱃지 연동
5. 뱃지 컬러는 기존 토큰 내에서, 카피는 모두 화법

## B. 양도 진행 가이드 — 관찰 가능 단계로 재구성
원칙: 앱이 DB로 판정할 수 없는 단계는 넣지 않는다. 계약서 작성·잔금 이전 완료 삭제.
새 6단계 (전부 자동 판정):
1. 매물 등록 — 매물 존재
2. 내부 사진 3장 올리기 — interior ≥3
3. 소개글 다듬기 — 모두 초안 저장·확정 이력 (구조상 판정 불가면 "소개글 확인" 탭 이력으로 대체, 어느 쪽인지 보고에 명시)
4. 매물 공개하기 — status = published
5. 첫 문의 받기 — D4 수신 ≥1
6. 가격 협의 시작 — status = negotiating 전환 또는 문의 첫 응답 발신 중 먼저 온 것
표시 규칙:
- 1~4: 다음 단계 CTA 유도 (현행 스타일)
- 5~6: "기다리는 단계" 시각 구분, CTA 없음, 카피 모두 화법 (예: "문의가 오면 모두가 바로 알려드려요")
- 6 완료 시 가이드 접힘 + "협의 진행 중" 한 줄 요약 (상세는 후속)
- 하드코딩 상태 금지 (과거 버그 재발 방지 테스트 포함)

## C. biz_type NULL 실매물 10건 — 재질문 확장
IndustrySubPrompt 노출 조건에 "업종 미입력(신·구 컬럼 모두 NULL)" 추가. 기존 팝업·저장 로직 재사용, 신규 UI 없음. 닫기 허용·재노출 현행 유지.

## D. 고아 대화 정리
conversations.listing_id가 존재하지 않는 매물을 가리키는 1건 — 삭제 SQL 제시 (A의 SQL과 함께 묶어 제시 가능, 실행은 대표 콘솔). 재발 방지가 코드로 간단하면 포함, 스키마 제약이 필요하면 제시만.
