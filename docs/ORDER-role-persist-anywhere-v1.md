# ORDER-role-persist-anywhere-v1 — 역할 선택 즉시 저장 (경로 무관 병합)

## 원칙 (대표 확정)
역할 선택은 선택한 그 순간 저장. 이후 로그인이 어떤 경로(A2 지름길, A3/A4 직접 진입, 게이트 발 가입 등)로 일어나든 finishLogin 병합에 반드시 합류. "다음 버튼에서 저장" 같은 내비 시점 저장 금지.

## 수정
1. A2: 칩 선택 토글 시점에 modu_pending_roles 즉시 갱신(다음/지름길 무관).
2. A3 각 축 화면: 진입 시 자기 축을 pending에 보장(A3Landlord 진입=landlord 보장). URL 직접 진입 커버.
3. A4: mode 무관하게 pending 읽어 병합 전달.
4. 로그인 상태에서 온보딩/A3 진입 시: 로그아웃 없이 해당 역할 즉시 roles 추가(서버 저장) 후 홈 복귀 — (+) 동선과 로직 공유.
5. (+) 버튼 실동작 점검: 추가 동선이 실제로 붙어 있는지, 4와 같은 로직인지.

## 중복 계정 조사 (보고만)
profiles에 김태우 seller 2개(하나 roles=["seller"], 하나 NULL). auth.users 대조 — 가입 방식·생성일·연결 listings 소유. 통합은 대표 결정 후 SQL.

## 테스트
경로 매트릭스: A2 지름길 / A3 직접 / A4 직접 / 로그인 상태 추가 4경로 병합. 전체 통과.
