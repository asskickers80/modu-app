# ORDER: 홈 매물 등록 CTA → 내 매물 카드 전환 (v1)

작성: 2026-07-20 채팅 세션. 구현은 Claude Code.

## 배경
- 현재 홈(A 축, 양도자)의 네이비 CTA "매물 등록하기"는 매물 등록 후에도 동일하게 유지됨
- 상위권 레퍼런스(당근마켓, 신상마켓 등): 등록 CTA는 매물 0건일 때의 온보딩 장치. 매물이 생기면 홈의 중심은 "내 매물 현황"으로 전환되고, 신규 등록은 보조 버튼으로 격하
- 이 오더로 대기 안건 "홈 CTA vs 더보기 '내 매물 수정하기' 역할 분담"도 함께 정리됨: 홈 카드 = 주 진입점, 더보기 시트 = 바로가기 보조 (더보기 쪽 변경 없음)

## 요건

### 1. 분기 조건
- 로그인/현재 페르소나 기준 본인 매물 수(status가 example·삭제 제외) 계산
- 0건: 현행 유지 — 네이비 "매물 등록하기" CTA 그대로
- 1건 이상: CTA 자리를 "내 매물 카드"로 교체

### 2. 내 매물 카드 (1건)
- 구성:
  - 대표 사진 썸네일 (내부 사진 1번). 사진 없으면 업종 카테고리 컬러 플레이스홀더
  - 제목: 상호 공개 시 상호명 / 비공개 시 "업종 · 지역" (예: "패스트푸드 · 경기")
  - 상태 뱃지: 공개중 / 협의중 등 현행 status 매핑
- 탭 → 해당 매물 상세로 이동 (수정은 상세에서, 현행 ?edit= 흐름 재사용)
- 카드 하단(또는 우측 상단) 텍스트 버튼 "+ 새 매물 등록" → /e1/1 진입. 중복 등록 허용 유지
- 카피는 모두 화법 준수 (ORDER-e1-copy-modu-voice 기준). "AI" 단어 금지

### 3. 2건 이상
- v1 범위: 카드에 "매물 N건" 요약 + 탭 시 내 매물 리스트로 이동 (가로 스와이프는 이번 범위 아님)
- 리스트 화면이 이미 있으면 재사용, 없으면 단순 리스트 신설

### 4. 가게 지표·문의 알림 섹션
- 이번 범위에서는 현행 위치 유지 (카드 흡수는 phase 2 후보로만 문서화, 구현 금지)
- 지표는 준비중 상태 유지 — 가짜 숫자 금지 (정직 원칙)

### 5. 제외 범위
- 더보기 시트 변경 없음
- D4 연동 없음
- 프리미엄/노출 차등 로직 없음

## 검증 기준
- Playwright: 매물 0건 계정 → CTA 노출 / 매물 1건 계정 → 카드 노출 + 탭 라우팅 + "+ 새 매물 등록" 라우팅 / 2건 계정 → N건 요약 + 리스트 진입
- example 매물만 있는 계정은 0건으로 취급되는지 확인
- 실기기(아이폰 390px) 확인 후 완료 처리
