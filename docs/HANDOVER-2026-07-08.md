## 카카오 로그인 — 구현 완료 상세

### 문제
- Supabase Kakao 프로바이더가 `account_email` 스코프를 서버에서 강제 추가
- 개인 개발자 앱은 `account_email` 권한 없음 → KOE205 에러

### 해결 방식
1. **A4SignUp.jsx** — `supabase.auth.signInWithOAuth` 제거, `window.location.href` 직접 리다이렉트
   - `scope=profile_nickname+profile_image` (account_email 없음)
   - `redirect_uri=${origin}/auth/kakao-callback`
2. **AuthKakaoCallbackPage.jsx** (신규) — 카카오 코드 → 토큰 교환 → 프로필 조회 → Supabase 계정 생성/로그인
   - Supabase 계정: `kakao_{id}@kakao.modu.internal` / `modu_{id}_kakao`
   - React Strict Mode 이중 실행: `sessionStorage('kakao_code_used')` 가드
3. **App.jsx** — `/auth/kakao-callback` 라우트 추가

### 카카오 콘솔 설정 (플랫폼 키 > REST API 키 수정)
- Redirect URI: `http://localhost:5173/auth/kakao-callback` (또는 cloudflared 터널 URL)
- ⚠️ cloudflared 재시작 시 터널 URL 변경 → 카카오 콘솔에서 업데이트 필요

### Supabase Auth 설정
- Kakao provider 활성화 (계속 필요 — 실제 OAuth 흐름은 우회하지만 설정은 유지)
- "Allow users without email" → ON

## 주요 버그 수정

### bizLabel 표시 오류 (A7SellerDashboard)
- 증상: 업종을 "기타"로 등록해도 대시보드 헤더가 "음식점 양도"로 표시
- 원인: `const bizLabel = myListings[0]?.biz_type` 가 `myListings` 상태 선언(line 150) 이전(line 130)에 위치 → Temporal Dead Zone이 아니라 값이 항상 undefined
- 수정: `primary` 선언 이후로 이동 → `const primary = myListings[0]; const bizLabel = primary?.biz_type ?? profile.bizType ?? '내 가게'`

### AuthCallbackPage 프로필 복원
- 기존 유저 재로그인 시 DB의 `nickname + profile_data`를 localStorage에 복원
- 이전엔 재로그인해도 닉네임이 복원되지 않는 문제
