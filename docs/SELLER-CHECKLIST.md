# 양도자(Seller) 기능 전수 체크리스트

> 코드 기준 전수 추출 (라우트·컴포넌트·onClick 근거, explorer 4개 병렬 조사).
> 형식: `[ ] 동작명 | 실행 방법 | 기대 결과 | 위험도`
>
> **위험도 기준**
> - `⚠️ <의존>` = 우리 Playwright 테스트가 **mock으로 가린 실환경 의존**이 걸린 항목
>   (Gemini 실호출 / Supabase Storage 실업로드 / Daum 우편번호 스크립트 / 실 DB RLS(insert·update·delete·select) / 국토부 실거래 API / 실 OAuth·Auth / Realtime)
>   → 실기기·실DB에서 별도 확인 필요.
> - `✅` = 클라이언트 로직만 (테스트로 실 DOM 검증됨 또는 외부 의존 없음)
