# 모두 업종 분류 체계 (INDUSTRY-CATEGORY-MAP)

온보딩·매물카드·계약서 등 모두 생태계 전체가 공유하는 단일 업종 분류.
설계 원칙: 화면 언어 = 사장님 언어 / 데이터 = 대분류+소분류+KSIC 코드 3계층 저장.
"빠지는 업종 없음" 보장: ① 모든 대분류에 '기타' 소분류 ② 업종 직접 검색은 전체 소분류 대상 + 검색 결과가 분류에 자동 매핑 ③ 최종 폴백 '기타(직접입력)'.

## 구현 규칙
1. **저장 구조**: category_main + category_sub + ksic_code 3필드. 직접입력 시 category_sub = 입력값, ksic_code = null
2. **대분류만 선택하고 넘어가기 허용** — 소분류는 선택 사항 (이탈 방지). 이 경우 category_sub = null
3. **업종 직접 검색**: 전체 소분류 + 동의어(예: "통닭"→치킨, "헤어샵"→미용실) 대상 검색, 선택 시 대분류·소분류 자동 세팅. 동의어 사전은 상수 파일에 확장 가능하게
4. **KSIC 코드 대조 완료 (2026-07-16, 10차 기준)**: 초안 대비 정정 — 노래방 91223 / PC방 91222 / 게임장(전자게임장) 91221 / 당구장 91135 / 볼링장 91134 / 헬스장(체력단련시설) 91132 / 사진관(인물사진 촬영) 73301. null 처리(단일 대표코드 불확정) — 뷔페(10차에 별도 코드 없음), 기타 음식점, 의류·패션(남/여/셔츠 등으로 세분·47416은 가죽·모피 소매), 기타 판매점, 요가·필라테스(시설운영 91132 vs 교육 85611 해석 갈림), 펜션·게스트하우스(민박 55104 vs 기타 55109 갈림), 병원·약국 임대(86은 대분류 코드). 참고: 모텔 단독은 여관업 55102(대표코드는 호텔업 55101 유지), 세차장 단독은 95213
5. 이 분류는 온보딩 전용이 아니라 **모두 전역 공통 상수**(categories.ts)로 구현 — 매물 등록, 검색 필터, 시세 데이터 조인에 동일하게 사용
