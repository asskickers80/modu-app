# ORDER-guest-access-v1 — 방문자 열람 자유 원칙 적용

## 배경
현재 방문자(비로그인)가 탐색·검색 진입 시 가입 게이트에 막힘.
이는 확정 설계 원칙 위반: 인증 게이트([F])는 행동 시점(토스 패턴)에만 걸린다.
핸드오버 v16 및 PROGRESS.md의 [공통] 누적 기록에 14번으로 추가할 것:
"방문자 열람 자유 원칙 — 탐색·검색·상세·콘텐츠는 비로그인 개방, [F] 게이트는 행동 시점에만."

## 원칙 (판정 기준)
비로그인으로 가능해야 하는 것:
- 매물 탐색, 검색, 매물 상세(E2) 열람
- 기업회원(업체) 검색·열람
- 커뮤니티/이야기 콘텐츠 열람 (인터뷰·사연·인사이트 포함)
- 기타 공개 콘텐츠 전부

[F] 게이트가 걸려야 하는 시점 (여기서만):
- 문의(DM) 발신
- 찜/저장
- 매물·글 등록
- 마이탭 등 프로필 필요 기능

## 구현 지시
1. 라우트 전수 조사: 가입/로그인 리다이렉트 또는 차단이 걸린 지점을 전부 목록화해서 먼저 보고할 것. (수정 전 목록 제시 → 진행)
2. 열람 계열 라우트에서 게이트 제거. 행동 계열은 버튼/액션 핸들러 시점에 게이트 유지.
3. 게이트 발동 시 UX: 차단 페이지로 튕기지 말고, 현재 화면 유지 + 가입 유도 시트/모달. 가입 완료 후 원래 하려던 행동 지점으로 복귀 (returnTo 보존).
4. 비로그인 상태에서 Supabase 조회가 RLS에 막히는 테이블이 있는지 확인. 있으면 코드 수정이 아니라 SQL 제시 후 멈출 것 (대표 콘솔 실행 원칙).
5. Playwright: 비로그인 fixture로 탐색→검색→E2 상세 진입 통과 테스트, 문의 버튼 클릭 시 게이트 시트 노출 테스트 추가.

## 금지
- 실 DB 검증 목적 쓰기 금지
- 스키마 변경 임의 실행 금지 (SQL 제시 후 멈춤)
- "합의된 보류" 임의 주장 금지
