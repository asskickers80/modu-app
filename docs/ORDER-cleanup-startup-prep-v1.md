# ORDER-cleanup-startup-prep-v1 — 소청소 + 창업준비 축 검토 준비

## 모드
자율 판정 모드 유지. 멈춤은 (a)스키마 SQL (b)실DB 정정 (c)판정 불가 3경우만.

## 1부: 소청소
1. /dev/supabase 임시 진단 세션 카드 제거. 완료 기준 = 라우트 사망 (코드 삭제 + 접근 시 404/홈 리다이렉트).
2. 인박스 안읽음 점 5곳 → components/UnreadDot.jsx 통일 (COMMON-APPLY-MATRIX 후속 ② 실행).
   판정 로직은 lib/unread.js isUnread 단일 소스 유지, 복제 금지.

## 2부: 창업준비 축 검토 준비 (대표 시뮬레이션 직전 정비)
3. 창업준비 온보딩→홈 진입 플로우 전수 점검: 암묵 기본값 함정(A4 seller 오설정류) 재확인 / 각 단계 뒤로가기·이탈 후 재진입.
4. 창업준비 홈(A7StartupFeed) 데모 카드 전수 목록화: 카드별 데이터 출처(실/데모/하드코딩) 표. 데모 카드는 아직 제거 금지 — (가)예시 뱃지 유지 vs (나)준비중 교체 두 방식 구현 지점만 파악해 보고.
5. 창업준비→탐색→E2 상세→문의 발신→양도인 수신까지 로그인 계정 기준 연속 동작 Playwright(D4 반대편 사이클). 게이트 미발동(로그인) 확인 포함.
6. 창업준비 화면 카피 점검: AI 라벨·권위 어휘 잔존 grep 재확인(매트릭스 기적용 검증).

## 산출물
- 데모 카드 목록표(4): 카드명 / 출처 / (가)안 구현지점 / (나)안 구현지점
- docs/SIM-STARTUP-v1.md: 창업준비 시나리오 순서(신규 이메일 가입→온보딩→홈→탐색→상세→문의→답장 수신→관심/찜) + 단계별 확인 포인트.
