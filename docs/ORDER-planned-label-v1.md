# ORDER-planned-label-v1 — 미구현 기능 (예정) 표기, 삭제 금지

## 모드: 자율 판정. 멈춤 (a)(b)(c)만.

## 원칙
장래 구현 예정이나 미구현인 기능·연출·표시는 삭제하지 않는다(전체 틀 보존).
문구 옆에 "(예정)"을 붙여 아직 동작하지 않음을 정직하게 표시. 기존 "(미구현)"도 "(예정)"으로 통일.

## 대상
1. E1p Step2 로딩 체크리스트: 실호출 없는 항목에 (예정)(위치·상권 수집, 등기·건축물 분석, 인근 임대 시세 비교 등). 실호출 항목(설명문 초안)은 표기 없이 유지. (예정) 항목의 완료 체크(✓) 연출 제거(안 한 일에 완료 표시 모순). 항목·레이아웃 유지.
2. E1(양도인) Step2 로딩도 동일 감사·규격.
3. 기존 (미구현) 표기 → (예정) 교체.
4. 전 축 grep: 미구현인데 동작처럼 표시된 지점 추가 발견 시 같은 규격(삭제 금지, (예정)). 목록 보고.

## 테스트: (예정) 표기 존재, (예정) 항목 완료 체크 부재. 전체.
