# 계약서 전자서명 앱 스펙 (CONTRACT-APP-SPEC)

## 개요
- 목적: 종이 광고계약서를 iPad 웹앱으로 대체. 필드 대부분이 기본값으로 채워진 상태에서 건별 정보만 입력 → 고객이 iPad로 확인·자필·서명 → PDF 생성 → **iPad 공유하기(share sheet)로 고객에게 즉시 전달** + 내부 자동 저장.
- 사용자: 내부 전용 (담당 에이전트 1인, 공개 배포 없음)
- 사용 기기: iPad Safari (PWA, 홈 화면 추가로 전체화면 실행)
- 스택: React + Vite + Tailwind CSS + Supabase (Storage, DB), pdf-lib
- 서버 발송 인프라 없음 (이메일/문자 미사용 — 공유 시트로 대체)

## 원본 양식: 점포라인 매물광고 이용계약서 (A청색NCR, 3차 수정약관)

### 1. 필드 구성 — "기본값 채워진 상태로 시작" 원칙

**건별 입력 (매번 새로 입력하는 4개) — 모두 키보드 입력 가능**
| 필드 | 타입 | 비고 |
|---|---|---|
| 상호 | text | 필수 |
| 업종 | 선택 + 직접입력 | 점포라인 업종 분류에서 2단계 선택(대분류→소분류). 목록에 없으면 키보드 직접입력. 부록 A 참조 |
| 사업자등록번호 | text | 숫자 10자리만 입력하면 자동으로 000-00-00000 하이픈 포맷 적용 (inputmode="numeric", 숫자 키패드) |
| 소재지 | text | 필수 |

**기본값 자동 완성 (수정 가능)**
| 필드 | 기본값 | 비고 |
|---|---|---|
| 담당 에이전트 | 저장된 이름 | 설정에서 1회 등록 |
| 광고상품명 | "광고" | 고정 |
| 광고료 / 부가세 / 총액 | 상품 선택 버튼으로 일괄 입력 | 아래 3종 |
| 광고개시일 | 오늘 (연도 2026 자동) | 날짜 표기 "2026년 X월 X일" — 원본 양식의 "20  년" 공란에 연도까지 자동 기입 |
| 광고기간 | 3개월 | 1/3/6/12개월 칩 선택, 기본 3 |
| 광고종료일 | 개시일 + 기간 | 자동 계산 |

**광고 상품 선택 (버튼 3개, 탭 한 번으로 금액 3종 필드 자동 입력)**
| 상품 | 광고료 | 부가세 | 총액 |
|---|---|---|---|
| 하단광고 | 1,000,000 | 100,000 | 1,100,000 |
| 중간광고 | 2,000,000 | 200,000 | 2,200,000 |
| 최상단광고 | 3,000,000 | 300,000 | 3,300,000 |

선택 후에도 금액 필드는 개별 수정 가능(수정 시 부가세 10%·총액 자동 재계산). 금액 표시는 콤마 포맷.

### 2. 고정 텍스트 (하드코딩)
- 전문 (광고 의뢰인/점포라인 계약 체결 문구)
- 제1조~제7조 전체 본문 (원본 기준 그대로, 밑줄/강조 유지)
- 광고료 입금계좌: 우리은행 1005-701-333816 예금주: ㈜점포라인
- 회사 정보: 주소 서울시 서초구 강남대로 617, 7층 / 상호 ㈜점포라인 / 대표이사 이상희
- 회사 직인 이미지 (투명 배경 PNG로 삽입)

### 3. 고객 자필 영역 — canvas 2개 (핵심)
1. **자필 확인란**: 중요내용 고지 확인. 고객이 "들었음"을 손글씨로 직접 작성. 확인 문구(구입 문의 부재/매매 불발 가능, 중도해지 시 10% 위약금+게재분 공제 후 14일 이내 환급)를 canvas 바로 옆에 표시.
2. **광고주 성명 + 서명(인)**: 성명은 텍스트 입력, 서명은 canvas.
- 구현: `signature_pad` 라이브러리 (터치/Apple Pencil 지원), 지우고 다시 쓰기 버튼 필수.
- 날짜: 서명 완료 시점 자동 기록 (표시 형식: 20XX년 X월 X일).

### 4. 화면 플로우 (3단계)
1. **작성**: 건별 4개 필드 입력 + 상품 프리셋 선택 → 미리보기 (나머지는 기본값 채워진 상태)
2. **고객 확인·서명**: iPad를 고객에게 전달. 계약서 전문 스크롤 열람(최하단까지 스크롤해야 서명 활성화) → 자필확인란 작성 → 성명 입력 + 서명
3. **완료 및 전달**:
   - PDF 생성 → Supabase Storage 자동 업로드 (저장 먼저, 전달은 그다음)
   - **"고객에게 보내기" 버튼 → iPad 공유 시트 호출** → 카카오톡/문자/메일/AirDrop 등 고객이 원하는 채널로 전달
   - **"광고료 결제하기" 버튼 → 점포라인 바로결제 페이지 오픈** (섹션 6 참조)
   - 완료 화면에 저장 상태 + 공유 완료 여부 표시

### 5. PDF 생성 및 공유
- pdf-lib로 클라이언트에서 생성 (원본 1페이지 레이아웃 재현)
- 서명/자필 canvas는 PNG로 추출해 해당 위치에 합성
- 파일명: `계약서_{상호}_{YYYYMMDD}.pdf`
- **공유**: Web Share API Level 2 사용
  ```js
  const file = new File([pdfBlob], fileName, { type: 'application/pdf' });
  if (navigator.canShare?.({ files: [file] })) {
    await navigator.share({ files: [file], title: fileName });
  }
  ```
  - iPadOS Safari 15+ 파일 공유 지원. HTTPS 필수 (Vercel/Netlify 배포로 충족)
  - 폴백: 공유 미지원 시 PDF 다운로드 → 파일 앱에서 수동 공유 안내

### 6. 점포라인 바로결제 연동
- 대상 페이지: `https://www.jumpoline.com/customer/payment.aspx` (유료 광고 결제 / 수수료 자유금액 결제, 이니시스 PG)
- **iframe 삽입 금지**: PG 보안 정책상 iframe 내 결제 불가. 반드시 `window.open(url, '_blank')`으로 새 창 오픈 (PWA standalone 모드에서는 in-app Safari 뷰로 열림 — 결제 후 앱 복귀 자연스러움)
- 결제 페이지는 금액 자동 전달(query param)을 지원하지 않으므로, **버튼 위에 총액을 크게 표시 + "금액 복사" 버튼** 제공 → 고객이 결제 페이지에서 붙여넣기
- 결제사유 입력용으로 상호명도 복사 버튼 제공 (예: "매물광고료 - {상호}")
- contracts 테이블에 `payment_opened_at` (결제 페이지 오픈 시각) 기록 — 결제 완료 여부는 점포라인 내부 시스템에서 확인 (앱에서 검증 불가)

### 7. 데이터 저장 (Supabase)
```
contracts 테이블
- id (uuid)
- 상호, 업종, 사업자등록번호, 소재지, 담당에이전트
- 광고료, 부가세, 총액, 개시일, 종료일, 기간
- signed_at (timestamptz)
- pdf_path (Storage 경로)
- device_info (증빙용: userAgent, 서명 시각)
```
- Storage 버킷: `contracts` (비공개, signed URL로만 접근)
- 목록 화면: 서명 완료 계약 검색(상호/날짜), PDF 재다운로드, **재공유 버튼**(Storage에서 받아 다시 공유 시트 호출)

### 8. 보안·법적 참고
- 접근 제한: 내부 전용이므로 간단 PIN 잠금
- 전자서명법상 당사자 서명 의사가 확인되는 전자서명은 효력 인정. 증빙 강화용으로 서명 시각·기기 정보 DB 기록
- 개인정보(사업자번호) 저장하므로 Storage/DB 비공개 설정 필수

### 9. 범위 제외 (1단계에서 안 함)
- 이메일/문자 자동 발송 인프라 (공유 시트로 대체 — 필요 시 추후)
- 다중 계약서 템플릿 (현재 양식 1종만)
- App Store 배포 (PWA로 충분)

## 부록 A. 업종 선택 목록 (점포라인 매물 검색 분류 기준)

2단계 선택 UI: 대분류 탭 → 소분류 버튼. 최근 선택한 업종 3개를 상단에 노출. 목록에 없는 업종은 "직접입력" 선택 시 키보드 입력.

**휴게음식점**: 카페, 치킨점, 커피점, 패스트푸드, 피자점, 제과점, 아이스크림점, 떡볶이/튀김, 기타휴게점

**일반음식점**: 한식점, 고깃집, 분식점, 중국집, 일식점, 생선회/해물, 레스토랑, 돈까스/우동, 기타음식점

**주류점**: 맥주호프점, 노래주점, 바, 이자카야, 실내포차, 꼬치구이, 기타주점

**오락스포츠**: 스크린골프장, 당구장, 노래방, 요가/필라테스, 골프연습장, PC방, 헬스클럽, 만화방, 기타오락스포츠

**판매업**: 편의점, 의류판매점, 1층다용도, 이동통신점, 슈퍼마켓, 아이스크림 할인점, 문구팬시, 화장품점, 기타판매점

**서비스업**: 독서실, 미용실, 마사지, 피부미용, 빨래방, 세차장/카센타, 네일아트, 키즈카페, 기타서비스업

**기타업종**: 상가매매, 모텔, 학원, 펜션, 병원/약국, 다용도상가, 기타부동산 ※ 이 대분류만 사이트 인코딩 문제로 일부 항목 추정 포함 — 구현 시 점포라인 검색 페이지에서 1분 대조 권장

업종 데이터는 별도 상수 파일(categories.ts)로 분리해 추후 수정 용이하게 구성.
