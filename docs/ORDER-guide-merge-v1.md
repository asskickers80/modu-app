# ORDER-guide-merge-v1 — 양도 가이드 6단계 → 5단계 (문의받기·협의시작 재편)

## 결정 (대표 확정)
"첫 문의 받기"와 "문의에 첫 답장"이 사실상 동시 점등되어 중복감 → 단계 통합.
새 구성 (5단계): 매물 등록 → 내부 사진 3장 → 소개글 다듬기 → 매물 공개 → 문의받기 → 협의시작
(위 나열은 6개로 보이지만 기존 5·6단계 두 칸이 문의받기·협의시작으로 재정의되는 것.
 즉 칸 수는 유지하되 의미 재편이 아니라, "첫 문의"+"첫 답장" 두 칸을 하나로 줄여 총 5칸.)

## 단계 정의 (관찰 가능 원칙)
5단계 "문의받기" = 첫 문의 수신 (기존 5단계와 동일 판정).
  - 탭 동작: 해당 첫 문의 스레드로 딥링크. 상세에 "언제·어떤 문의"가 보이도록
    (완료 상태 서브텍스트에 첫 문의 일시 표시, 예: "7월 21일 첫 문의 도착").
6단계 → "협의시작" = 소유자 첫 답장 발생 (ownerReplied, 관찰 가능).
  - 양방향 대화 성립 = 협의 시작으로 정의.
  - 탭 동작: 메시지함(인박스).
  - status=negotiating 전환은 별도 개념으로 유지 (탐색 뱃지·접힘 카드는 기존대로 negotiating 전용).

## 구현 지시
1. 가이드 단계 배열·판정·라벨 수정. 미답장 문의가 있으면 "협의시작" 칸에 유도 문구
   (예: "답장을 기다리는 문의 N건") — 모두 화법, 권위 어휘 금지.
2. 완료 단계 탭 내비게이션 유지 (전 단계 탭 진입 원칙). 문의받기 딥링크 라우팅 실동작 확인.
3. 이 단계 구조는 [공통] 후보이므로, 다른 프로필 가이드에도 적용 가능한 형태로
   단계 정의를 데이터/설정으로 분리할 수 있으면 분리 (무리한 추상화는 금지, 판단 보고).
4. Playwright: 5단계 점등 순서(수신→답장), 딥링크 라우팅, 미답장 유도 문구 표시/해제 검증.
