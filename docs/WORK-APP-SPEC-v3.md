# 전용 업무 앱 스펙 v3 (WORK-APP-SPEC-v3)

> v2 → v3 개정 (2026-07-09, 대표님 결정 사항 반영):
> - 2번 탭 '상담 메모'(화면 캡처 + 텍스트 포스트잇 + 인트라넷 중계 모드)를 공식 기능으로 추가
> - 캡처 버튼 이름을 '반영' → **'캡처'**로 변경. **'반영'은 매물카드→수수료 계산 버튼 전용 용어**
> - 손글씨 스티커(획 JSON)는 매물카드 구현 후 매물카드 위에 얹는 것으로 순서 명시
> - 탭 구성·순서 확정 (아래)

상단 탭 6개로 확장하며, **탭을 하나씩 순차 구축**한다.
기존 CONTRACT-APP-SPEC(계약서 전자서명)을 계약 단계로 흡수하는 상위 앱 확장.
기존 계약서 코드는 수정 없이 유지하고, 앞단에 고객·매물 관리 층을 추가한다.

## 탭 구성 (확정)

```
[1 천하통일] [2 상담 메모] [3 매물카드] [4 계약서] [5 노트] [6 준비 중]
```

- 배치 근거: 실제 업무 흐름(인트라넷 조회 → 상담하며 캡처·메모 → 고객/매물 기록 → 계약)이
  왼쪽→오른쪽 순서와 일치. 계약서는 기존 4번 위치 유지(사용 습관 보존). 노트는 보조 도구로 5번.
- 구현 현황: 1·2·4번 구현 완료, 3번 진행 중, 5·6번 준비 중

## 전체 플로우 (6단계)
```
① 앱 로그인 → 인트라넷 → ② 캡처/상담 메모 → ③ 매물카드(전화번호 조회)
→ ④ 반영(수수료 계산) → ⑤ 계약 및 결제(계약서 자동 주입) → ⑥ 서명·PDF·공유·바로결제(기존)
```

## ① 로그인 및 인트라넷 (1번 탭 '천하통일') — 구현 완료
- 앱 자체 잠금: PIN 로그인 (기존 스펙 유지)
- 로그인 직후 화면: 인트라넷 탭 (기본 첫 화면)
- **동작 모드 2가지**:
  - **중계(프록시) 모드 — 공식**: `.env`의 `INTRANET_TARGET` 설정 시 개발 서버가 인트라넷을
    같은 출처(`/`)로 중계. 앱은 `/app/` 경로. 같은 출처가 되어 **캡처**가 가능해진다.
    자격증명은 여전히 브라우저↔인트라넷 직접 처리 (앱은 관여하지 않음)
  - 직접 모드(폴백): 주소를 앱에서 입력해 iframe 직접 임베드. 캡처 불가.
    임베드 차단 시 "인트라넷 열기" 새 탭 버튼
- **다중 창 (천하통일 탭 내부)** — 구현 완료:
  - 탭 바 아래 창 칩 스트립: (창1) (창2) ... [+], 탭하면 전환, X로 닫기(활성 칩에만 표시), 길게 눌러 이름 변경
  - 각 창 = 독립 iframe, 전환 시 숨김 처리만(unmount 금지) → 스크롤·입력·세션 상태 유지
  - 창 개수 상한 5개 (메모리 보호)
  - 알려진 한계: cross-origin이라 창 제목 자동 표시 불가(수동 이름), 인트라넷 popup(window.open)은 스트립으로 포섭 불가 — popup 다용 시 임베드 방식 재검토
- **[캡처] 버튼** (툴바, 중계 모드 전용): 현재 창 화면 전체를 이미지로 캡처 → 2번 탭 '상담 메모'로 전송

## ② 상담 메모 (2번 탭) — 구현 완료 (v3에서 공식화)
- 1번 탭에서 [캡처]한 화면이 고정 배경으로 표시
- **텍스트 포스트잇**: [+ 메모 붙이기] → 추가, 드래그 이동, 탭하여 수정, ✕ 삭제, 색상 로테이션
- 저장: 현재 IndexedDB(로컬). **Supabase 이전은 매물카드 스키마 적용 때 함께 진행**
- 손글씨 스티커(획 JSON, perfect-freehand, Apple Pencil)는 **매물카드 구현 후
  매물카드 위에 얹는 기능으로 진행** (아래 ⑤ 메모 섹션)

## ③ 매물카드 (3번 탭, 고객·매물 관리) — 진행 중
- 진입: 전화번호 입력 (숫자 키패드, 010-0000-0000 자동 포맷)
  - 기존 고객 → 매물카드 즉시 오픈
  - 신규 → 새 카드 생성 화면
- 카드 기본 정보:
  - 고객: 전화번호(key), 고객명, 구분(양도자/임차인/기타)
  - 매물: 상호, 업종(기존 업종 선택 UI 재사용), 사업자등록번호, 소재지
  - 거래 조건: 보증금, 월세, 권리금, 관리비
- 목록 화면: 최근 상담순 정렬, 전화번호/상호 검색
- 저장: Supabase (customers/listings — 스키마 SQL은 별도 파일 제시, 콘솔 실행은 대표님)

## ④ 반영 → 수수료 계산 화면 — 미착수
- **'반영' = 매물카드에서 수수료 계산 화면으로 가는 버튼 (이 용어는 여기 전용)**
- 매물카드에서 [반영] 버튼 → 계산 결과 화면 로딩
- 자동 계산 항목:
  - **권리금 수수료**: 권리금 × 3~5% (VAT 별도) — 3/4/5% 슬라이더 또는 버튼, VAT 포함가 병기
  - **부동산 중개보수**: (보증금 + 월세×100) × 0.9% (VAT 별도)
  - 광고료: 하단/중간/최상단 3종 상품가 표시 (기존 프리셋 재사용)
- 고객에게 iPad 화면을 그대로 보여주며 상담하는 용도 → 큰 글씨, 표 형태
- 계산 결과는 카드에 스냅샷 저장 (계산 시점, 적용 요율 포함)

## ⑤ 메모 — 스티커형 손글씨 주석 (매물카드 구현 후 진행)
- 매물카드 화면 어디서든 [+메모] → 화면 중앙에 손글씨 캔버스 확대 오픈
- **펼침 상태 (작성 모드)**
  - Apple Pencil 필기: perfect-freehand 등 필기 전용 렌더링
  - 팜 리젝션: `pointerType === 'pen'`만 획으로 인식, 터치는 스크롤/제스처로
  - 도구: 펜 색 2~3종, 지우개(획 단위), 전체 지우기, 텍스트 입력 모드 전환
- **접힘 상태 (스티커)**: [완료] 시 축소 → 카드 위에 스티커 부착(미니어처 썸네일),
  드래그 자유 배치(위치 저장), 카드당 여러 장, 탭하면 재확대·재편집
- **저장 구조**: PNG 스냅샷이 아닌 **획(stroke) 데이터 JSON 저장** → 재편집 가능
  - memos 테이블: `strokes (jsonb)` + `thumbnail_path` + `pos_x, pos_y` + `updated_at`
  - 저장 시점: [완료] 버튼 + 30초 자동 저장

## AI 기능 층 (5번 탭 '노트' 및 요약 기능) — 미착수
- **인프라**: Anthropic API 사용. API 키는 클라이언트에 노출 금지 — Vercel Serverless Function(/api/ai)을 프록시로 두고 키는 환경변수 보관. 모델은 구현 시점의 최신 문서 기준으로 선정
- **요약 기능**: 인트라넷 자동 크롤링 불가 → 붙여넣기 기반.
  큰 텍스트 붙여넣기 영역 + [요약] 버튼 + 프리셋 3종(매물 목록 → 조건 표 정리 / 공지·문서 → 3줄 요약 / 자유 요약). 결과 복사 버튼, 필요 시 노트로 저장
- **노트 탭 (대화형 노트)**: 노트 = 대화 스레드. 메시지 입력 → Claude 응답 → 전체가 노트로 저장.
  notes 테이블: id, listing_id(FK, nullable), title, messages(jsonb), updated_at.
  매물카드에서 [노트] 진입 시 카드 정보(상호, 조건)를 대화 컨텍스트로 자동 포함.
  시스템 프롬프트: 상가 창업·점포 거래 상담 보조 컨텍스트 고정

## ⑥ 계약 및 결제 → 계약서 자동 주입 — 미착수 (계약서 플로우 자체는 구현 완료)
- 매물카드에서 [계약 및 결제] 버튼 → 기존 계약서 입력 화면(4번 탭)으로 이동
- **자동 주입 필드**: 상호, 업종, 사업자등록번호, 소재지 (매물카드 → 계약서 폼)
- 계약 완료 시 contracts 레코드에 customer_id 연결 (카드에서 계약 이력 조회 가능)
- 이후 단계는 CONTRACT-APP-SPEC 그대로: 계약서 설명 → 자필 "들었음" → 성명·서명 → PDF 생성 → Storage 저장 → 공유 시트 → 바로결제

## 데이터 모델 추가 (Supabase)
```
customers
- id (uuid), phone (unique, key), name, type(양도자/임차인/기타), created_at

listings (매물카드)
- id, customer_id (FK)
- 상호, 업종, 사업자등록번호, 소재지
- 보증금, 월세, 권리금, 관리비
- updated_at (최근 상담순 정렬용)

calculations (수수료 계산 스냅샷)
- id, listing_id (FK), 권리금수수료율, 계산결과(jsonb), created_at

memos (스티커 메모)
- id, listing_id (FK)
- type (handwriting/text)
- strokes (jsonb — 획 데이터, 재편집용), content (text 메모용)
- thumbnail_path (접힘 스티커 표시용 PNG, Storage)
- pos_x, pos_y (카드 내 스티커 위치)
- created_at, updated_at

contracts (기존 테이블에 컬럼 추가)
- + customer_id (FK, nullable — 단독 계약도 허용)
```
- RLS: 전 테이블 소유자 단독 접근 (개인용 앱)

## 개인정보 유의
- 고객 전화번호·사업자번호를 개인 Supabase에 저장하게 됨 → Storage/DB 비공개 필수, 앱 PIN 잠금 필수
- 회사 고객 데이터의 개인 인프라 보관에 대한 사내 정리(퇴사 시 데이터 이관/파기 포함)는 별도 판단 사항
