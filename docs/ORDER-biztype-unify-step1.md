# ORDER: 업종 분류 통일 — 1단계 (categories.ts 가동)

## 확정 판단 (2026-07-20 대표 결정)
① 매핑 불가 7종: 안경 소분류 신설 (개체수 많은 업종). 인력파견·운송·이사 등 비점포 업종은 소분류 신설 없이 해당 대분류만 저장 (소분류 NULL 정상)
② 평면 12종 병합 8건 해당 기존 매물(실측 6건): 소유자 다음 접속 시 재질문 — "업종을 새 분류로 확인해주세요" 칩 선택 1회. 응답 전까지 소분류 NULL
③ E1 업종 입력: A3의 대분류→소분류 드릴다운 + 동의어 검색 컴포넌트 재사용
④ 완성도 점수(lib/completeness.js)에 업종 입력 항목 추가. 필수 입력화는 하지 않음

## 구현 순서
1. categories.ts에 안경 소분류 추가 → BIZTYPE-UNIFY-STEP0.md 매핑표를 확정판으로 갱신
2. SQL 갱신 제시 → 멈춤 → 대표가 Supabase 콘솔에서 백필 실행 → 대표 확인 후 계속
3. E1 업종 입력을 A3 컴포넌트 재사용으로 교체, 신규 컬럼에 저장 (프랜차이즈 선택 시 brands 매핑표로 자동 세팅)
4. 조용히 깨진 3곳(ExplorePage 같은 업종 필터, E1Step4 시설 추천, collect-market-news 뉴스 매칭)을 신규 컬럼 기준으로 정정
5. 홈 헤더 업종 표기를 신규 컬럼 기준 "대분류 > 소분류"로 통일 (소분류 NULL이면 대분류만)
6. 재질문 플로우: 대상 매물 소유자 홈 진입 시 1회 노출, 칩 선택 즉시 저장. 무시(닫기) 허용 — 다음 접속 재노출
7. 완성도 채점에 업종 반영

## 주의
- 기존 컬럼 삭제 금지 (병행 유지, 정리는 별도 오더)
- 카피는 모두 화법, "AI" 단어 금지
- 재질문은 강제 게이트 아님 (닫기 가능)
