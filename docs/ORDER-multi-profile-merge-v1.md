# ORDER-multi-profile-merge-v1 — 멀티 프로필 병합 결함 (조사 후 수정)

## 모드: 자율 판정. 멈춤 (a)(b)(c)만.

## 증상 (실기기, 프로덕션)
비로그인 온보딩 임대인 선택 → 기존 계정(양도인+운영중 보유) 로그인 → 홈 상단에 양도인만.
온보딩 선택(임대인) 소실, 기존 운영중 미표시. 상단 칩 옆 회색 점·(+) 동작 불명.

## 조사 (수정 전 원인 보고)
1. 프로필 저장 구조: category 단수 컬럼 vs 복수(배열/행). v13=멀티프로필인데 단수면 구조 결함.
2. 로그인 병합: syncCanonicalDeviceId/mergeDeviceData/finishLogin이 온보딩 선택+계정 기존을 병합 vs 덮어쓰기. 생존 값 추적.
3. 상단 칩 UI: 렌더 소스, 회색 점·(+) 의도.
4. 대표 계정 실데이터: 읽기 전용 SQL 제시.

## 수정 방향 (구조에 따라)
- 단수면 복수 구조 전환 SQL 제시 후 멈춤(a).
- 병합: 온보딩 선택 + 계정 기존 = 합집합. 덮어쓰기 금지.
- 칩: 보유 프로필 전부 렌더+탭 전환. (+)=해당 축 A3만 진입(전체 온보딩 재시작 금지).
- 활성 프로필=기기 로컬, 프로필 목록=서버.

## 테스트·보고
합집합/전환/추가 + 전체 스위트. 4줄(ls-remote)+원인 요약+구조 판정+4번 SQL.
