# ORDER-auth-session-debug-v1 — 로그인 세션 미인식 진단 (수정 전 원인 보고만)

## 증상 (실기기, 프로덕션)
가입/로그인을 완료해도 앱 어디에도 로그인 상태가 없음. 문의 게이트도 계정 없음으로 판정.
(카카오 로그인 알림은 수신됨 = 카카오 인증 자체는 발생)

## 조사 항목 — 어디서 끊기는지 체인 특정
1. OAuth 콜백: 카카오 인증 후 리다이렉트가 앱으로 정상 복귀하는가.
   Supabase redirect URL 설정에 프로덕션 도메인(modu-app-asskickers80s-projects.vercel.app)이
   등록돼 있는가. localhost만 등록된 것 아닌지 — 유력 후보.
2. 콜백 후 세션 수립: supabase.auth.getSession()이 유효 세션을 반환하는가.
   detectSessionInUrl / 콜백 처리 코드가 프로덕션 라우팅에서 실행되는가.
3. 세션 지속: 로그인 성공 가정 시 localStorage에 sb-* 토큰이 남는가, 새로고침 후 유지되는가.
4. 앱 소비: 세션이 있어도 앱 상태(user)로 읽어오는 코드가 연결돼 있는가.
   syncCanonicalDeviceId는 호출되는데 user 상태 반영이 누락된 것은 아닌지.
5. 이메일 가입 경로도 동일 체인 점검 (카카오와 끊긴 지점이 같은지 다른지).

## 보고
체인 중 끊긴 지점 특정 + 근거(설정 화면/코드 경로/콘솔 로그). 수정은 원인 보고 후.
Supabase 대시보드 설정 확인이 필요하면 확인할 항목을 대표에게 요청할 것.

## 금지
실 DB 쓰기 검증 금지 / 인증은 무인 작업 금지 원칙 — 수정 단계는 대표와 함께 진행.
