# ORDER-inquiry-alert-v1 — 홈 문의 지표 재편 (새 문의 강조 + 누적 서브)

> ORDER-guide-merge-v1의 추가 항목 5를 이 오더로 대체·구체화.

## 결정 (대표 확정)
1. "진지도" → "진척도"로 명칭 변경, 표시는 "준비중" 유지 (로직 미구현 상태 정직 표시).
2. "문의" 카운트 칸과 "새 문의 알림"을 하나로 통합:
   - 주 표시 = 새 문의(미확인) 수. 가장 눈에 띄게 (색·크기로 강조, 0이면 강조 해제).
   - 부 표시 = 누적 문의 수. 비활성 톤으로 뒤에 (예: "새 문의 1 · 전체 2").
3. 새 문의가 이 카드의 최우선 정보다. 미확인 존재 시 한눈에 들어와야 함.

## 판정 설계
"미확인" = 소유자가 해당 스레드를 아직 열지 않은 문의.
- 관찰 가능해야 함. 현재 읽음 상태 컬럼이 없으면:
  conversations에 owner_read_at (timestamptz null) 추가 제안 → SQL 제시 후 멈춤 (대표 콘솔 실행).
- 판정: 스레드에 문의자 발신 메시지가 owner_read_at 이후(또는 read_at null)에 존재 → 미확인.
- 소유자가 D4 스레드 진입 시 owner_read_at 갱신.
- 메시지 탭 뱃지도 같은 판정 소스 공유 (단일 소스 원칙, 복제 금지).

## UI
- 탭 동작: 미확인 1건이면 해당 스레드 딥링크, 2건 이상이면 인박스(미확인 필터).
- 문구는 모두 화법. 권위·재촉 어휘 금지.
- 진척도 칸은 기존 준비중 처리 유지, 명칭만 교체.

## 테스트
Playwright: 미확인 발생→강조 표시, 스레드 진입→해제, 누적 수 유지, 0건 비강조. 참·거짓 양쪽 검증.
