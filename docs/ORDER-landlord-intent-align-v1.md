# ORDER-landlord-intent-align-v1 — 임대인 홈 의도 개인화 (A3 답변 기준)

## 모드: 자율 판정. 멈춤 (a)(b)(c)만.

## 원칙 ([공통])
A3 답변 = 홈 개인화의 진실의 원천. 실등록 객체 생기면 객체 기준으로 승격.
임대/매각은 사람 속성이 아니라 상가별 상태 — 홈 어휘는 답변/실상가를 따르고, 상가별 구분은 카드 뱃지가 담당.

## 헤더 (A3 상태 답변 → 등록 후 실상가 승격)
- 공실 채우기: "○○ 상가 임차인 찾는 중"
- 매각: "○○ 상가 매각 준비 중"
- 둘 다: "상가 임대·매매 준비 중"
- 등록 1개+: 실상가 deal_type 집계 승격 — "상가 N개 · 임대 N · 매각 N"
  (1개면 단수, both 상가는 "임대·매매" 표기 Code 자율)

## 가이드
제목 "상가 진행 가이드" 고정. 단계 어휘가 의도 추종:
공실형 "임차 문의받기" / 매각형 "매수 문의받기" / 둘다·혼재 "문의받기".
의도 판정: 등록 상가 있으면 실 deal_type, 없으면 A3 답변.

## 완성도
"내 상가 정보 완성도"로 개칭(중립 고정). 준비중 유지.

## 빈 상태 CTA
공실형 "상가 등록하고 임차인 찾기" / 매각형 "상가 등록하고 매수자 찾기" / 둘다 "상가 등록하기".

## 전수 감사
임대인 축 전체(홈·E1p·E2L·인박스·빈상태) grep — "임대" 단독이 매각 의도 배제하는 지점 목록화 → 중립화/의도추종 교체. 전후 목록 보고. "상가 임대 관리" 헤더 포함.

## 테스트
의도 3종(공실/매각/둘다)×등록 전후 헤더·가이드·CTA 매트릭스. 전체 스위트.
