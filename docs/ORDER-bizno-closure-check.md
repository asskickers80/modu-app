# ORDER: 사업자번호 등록 + 폐업 자동 감지 (주 1회)

## 배경
매물의 사업자 폐업을 국세청 API로 관찰해 데이터 정확성 유지. 원칙: 폐업 ≠ 양도 완료 (양도 성사 폐업 / 실패 철회 / 시설·집기 양도 지속 시나리오가 갈림) — 따라서 자동 완료 금지, 자동 비공개 + 소유자 확인.

## 요건

### 1. 등록 시 사업자번호
- E1에 사업자번호 입력 추가. 국세청 진위확인 통과 시에만 공개 가능 ([F] 게이트 철학 — 공개 직전 시점에 검증, 온보딩 부담 금지)
- 기존 published 매물 중 사업자번호 없는 건 처리: 소급 강제 금지, 다음 수정 진입 시 요청 (비공개 강등 없음 — 기존 사용자 신뢰 우선)
- 번호는 비공개 정보 (방문자 노출 금지). 스키마 변경 필요 시 SQL 제시 후 멈춤

### 2. 주 1회 배치 (크론)
- published·negotiating 매물 대상 국세청 상태조회 — 주 1회 (뉴스 수집 크론과 같은 인프라 패턴)
- 폐업 감지 시: 즉시 hidden 전환 + closure_detected 기록 (시각·이전 status)

### 3. 소유자 확인 요청
- 폐업 감지된 매물 소유자 홈 진입 시 확인 카드: "사업자 폐업이 확인됐어요. 어떻게 할까요?" (모두 화법)
- 3택: ① 양도 완료했어요 → completed ② 시설·집기 양도로 계속 → 재공개 (양도방식 표기 갱신) ③ 매물 내리기 → hidden 유지
- 무응답 시 hidden 유지 (폐업 매물 공개 방치만은 차단)

### 4. 비용·한도
- 국세청 API 호출량 확인 후 배치 묶음 크기 설계 (현 13건 기준 일단 단순 순회로 충분)

## 검증
- Playwright: 진위확인 게이트 / 폐업 mock → hidden 전환 / 확인 카드 3택 각 경로
- 크론 실행 로그 + 상태 전환 카운트 증빙
- 실기기 확인 후 완료
