# ORDER-address-real-everywhere-v1 — 주소 입력 전수 실구현

## 모드: 자율 판정. 멈춤 (a)(b)(c)만.

## 지시
앱 전체(전 프로필·전 화면) 주소 입력/검색 지점을 전수 조사, 양도인 E1의 실구현(Daum Postcode) 기준으로 실동작 교체.

## 작업
1. 전수 조사: 주소 검색·입력·지역 선택 모든 지점 grep(E1p·창업/운영/기업 온보딩·프로필 지역·A3 축별 지역·탐색 필터 등) 목록화.
2. 분류: (가) 상세 주소 필요(등록) → Daum Postcode / (나) 시도·시군구면 충분(온보딩·탐색 필터) → 기존 칩 실동작이면 유지, 더미면 실구현. 근거 표기.
3. 공통 컴포넌트: E1 주소 입력 → components/AddressInput 추출, (가) 전부 재사용. 복제 사망 확인.
4. 저장 매핑: 각 지점 주소 → 테이블 필드(address·region·sidoFromAddress) 정상 저장 확인. 저장처 없으면 [저장처 없음] 표기(교체하되 저장 연결 유보).
5. E1p "지도" 탭: 실구현 없으면 제거.
6. 더미 주소검색 껍데기 전부 제거(완료 = 옛 코드 사망).

## 테스트: (가) 지점 각각 주소 선택→값 반영, E1 회귀, 전체.
