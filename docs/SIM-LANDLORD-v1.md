# SIM-LANDLORD-v1 — 임대인(소유주) 축 실기기 시뮬레이션 브리프

> 돌려보며 잡기. 각 단계에서 **[미결 #N]**이 걸리면 그 자리에서 대표 결정.
> 미결 원문은 `docs/LANDLORD-PLAN.md` (c)절 7개 질문. 정식 주소: https://modu-app-asskickers80s-projects.vercel.app

## 시나리오 순서 & 확인 포인트

### 1. 신규 이메일 가입
- A2(구름) → **"소유주"(임대인)** 선택 → 다음 → A3(임대인 질문) → A4 → **이메일 회원가입**.
- ✅ 로그인 유지(로그아웃 안 튕김), 마이 ⋯ 하단 로그아웃 노출.
- ✅ 역할이 **임대인(landlord)로 유지**(seller 오설정 없음 — `A3Landlord:199 category:'landlord'` 확인됨).

### 2. 임대인 온보딩 (A3)
- 3문항: **상가 위치 · 상태(공실/매각/둘다) · 개수(1/2~3/4+)**.
- ✅ 각 단계 뒤로가기(←, `navigate(-1)`) 정상, 이탈 후 재진입.

### 3. 홈 (A7LandlordDashboard)
- ✅ "오늘의 한 마디" · "모두가 보는 임대 시세" — **AI 글자 없음**(기적용 확인).
- ✅ 자산 현황·자산별 카드·시세 = **전부 "준비중"(ComingSoon)** — 가짜 수치 없음.
- ⚠️ **[미결 #6]** — "내 공개 상가 카드" 필요 여부 미정 → 현재 자산 카드는 준비중 자리표시만.
- ⚠️ **[미결 #3]** — 완성도 점수 항목/배점 미정 → 완성도 카드 준비중.
- ⚠️ 진행 가이드(GUIDE_STEPS)는 **하드코딩 더미**(상가 0건인데 '상가 등록 done:true') — 설계 의존이라 미수정.

### 4. E1p 상가 등록 — 임대 분기 (1회)
- 홈 "상가 등록"(→ /e1p/1). 1단계 상단 **임대/매각/둘다 분기** 선택 → **"임대"**.
- ✅ 임대 조건(보증금·월세·관리비), 권장 업종(임대일 때만) 입력 UI 동작.
- ⚠️ **[미결 #1]** — 단계 수(현재 5단계, 고령 UX상 3단계 축소 검토) 미정.
- ⚠️ **[미결 #2]** — 2단계 AI 초안(Gemini) 넣을지 미정 → 현재는 생성 단계 존재.
- ⚠️ **[미결 #4/#5]** — E1pStep5 "공개"해도 **Supabase 저장 없음**(`E1pStep5`는 확인 후 `/a7/landlord`로 navigate만) → 등록이 **영속되지 않음**. 저장 위치(listings 재사용 vs 별도 테이블)·device_id 소유권 미정.

### 5. E1p 상가 등록 — 매각 분기 (1회)
- 다시 /e1p/1 → **"매각"** → 매각 조건(매매가·캡레이트=연 임대수익÷매매가×100) 입력.
- ✅ 매각 분기 UI 동작. 단 4번과 동일하게 **저장 안 됨** **[미결 #4]**.

### 6. E2L 상세 확인
- 탐색/홈에서 상가 상세(/e2l/:id) 진입.
- ✅ **비로그인 열람 가능**(게이트 없음 — App 라우트에 가드 없음).
- ✅ "동종 월세 시세" note = **"모두 추정"**(AI 라벨 제거됨).
- ⚠️ 상세는 **하드코딩 샘플 상가**(E2L `LISTINGS`) — 실등록분 아님. **소유자 모드 없음**(device_id 없음) **[미결 #4/#5·#6 관련]**.

### 7. 타계정 문의 → 수신 확인
- E2L 하단 "임대인에게 DM 문의하기" → **"빈 점포 DM 문의는 준비 중이에요 🚧" 토스트**.
- ❌ **현재 임대인 앞 문의는 도착 불가** — E2L DM이 Supabase 미연결(conversation 생성 안 됨). 양도인 축과 달리 반대편 수신 사이클이 아직 없음.
- (참고) D4LandlordInbox 자체는 conversation 조회·발신자 판정(viewerIsInquirer) 공유로 준비됨 — 실 conversation이 생기면 바로 동작.
