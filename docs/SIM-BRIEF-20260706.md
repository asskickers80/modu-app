# 고객 시뮬레이션 브리핑 — 양도자 첫 진입 (2026-07-06)

**시작 팁: 시크릿 창(또는 안 쓰던 브라우저)으로 시작.** 로그인이 없어서 기기(device_id)가 곧 사용자 — 새 창이어야 진짜 신규 고객 상태.

## 진입·가입
- **카카오 로그인은 아직 안 됨** (90% 완성, KOE205 — email scope 제거로 풀리는 건 확인됨, 미적용). 휴대폰 가입 흐름은 코워크가 통과했으니 될 것.
- 본인인증·사업자등록증 확인은 **"준비중"이 정상** (완료 배지 모순 제거됨).

## E1 등록 — 핵심
- **Daum 주소 팝업 직접 타이핑이 최우선 확인 지점** — 코워크가 유일하게 우회한 곳, 실사용 경로 미검증.
- 주소 선택 후 층·면적 자동채움 **없는 게 정상** — "건축물대장 자동조회 준비중" 안내가 떠야 함 (가짜 자동채움 제거됨).
- **사진 업로드 → E2 표시**는 미검증 항목. Step5 체크리스트가 사진 올린 것만 완료로 바뀌는지 확인 (수정된 버그 — 원래 전 항목 무조건 완료였음).
- AI 초안은 6~11초 소요 (ModuSpinner 돌면 정상, 503 나면 재시도 한 번). "검수"는 AI가 아니라 사용자 선택 동작.
- 새로고침해도 입력이 살아있어야 정상 (임시저장), 제출하면 지워져야 정상.
- **예시✦는 실등록 아님** — example 상태로 저장, 탐색 미노출. 상호명·주소를 실값으로 바꾸면 실등록 승격. 단 **예시 등록 후 수정-저장해도 example에 머묾** (승격 UX 미결 — 대표님 결정 대기).

## A7 대시보드
- 실데이터: 완성도, 내 매물 카드, AI 코칭(일일 캐시), 숨기기/거래완료 버튼.
- "준비중"이 정상: 매출·조회수·시장동향·거래처 등 수치 전부.
- 새 문의 배지는 실연결(DB) — DM 오면 진짜로 떠야 함.

## D4 문의·연락처 교환
- DM 자체는 실연결. **단 연락처 교환의 "상대방 수락"은 더미 버튼** — 상대가 실제로 수락하는 흐름이 없어서, 시뮬레이션 때 본인이 눌러야 완료 카드가 뜸. 현재 최대 미구현 구멍.

## 시뮬레이션과 무관하지만 알아둘 것
- RLS는 여전히 열려 있음 (UPDATE 미차단 실증 — 로그인 도입 시 해결, 의도된 한계).
- **NOT NULL SQL 콘솔 실행 — 아직이면 시작 전에.**
- 창업 피드의 빈 점포 카드(E2L)는 더미 상세로 연결됨 — 양도자 동선엔 안 걸리지만 들어가도 놀라지 말 것. 임대인 축 결정과 묶임.
