# ORDER-e1p-yield-required-v1 — 매각 상가 수익률 필수화 (임차 현황 분기)

## 모드: 자율 판정. 멈춤 (a)(b)(c)만.

## 결정
매각(sale·both) 수익률 필수. 임차 현황 분기:
- 임차인 있음 → 실계약: 현 보증금·월세(필수) → 수익률 자동 계산
- 공실 → 예상: 예상 보증금·월세/시세(필수) → "예상 수익률" 구분 표기

## 구현
1. E1pStep1 "현재 임차인이 있나요?" 칩 2택 [있어요/공실이에요] = occupancy 저장. 임대 단독도 유효(공실=입주 가능 시점), 필수 강제는 매각·둘다만, 임대 단독 필수 여부 Code 자율+근거.
2. 스키마: occupancy('occupied'/'vacant')·현/예상 보증금·월세 — 기존 컬럼 재사용 우선, 신설분 SQL 제시 후 멈춤(a).
3. 캡레이트 = 연 임대수익/매매가. 실계약 "수익률"/공실 "예상 수익률" 라벨. E2L 동일 구분(오인 금지).
4. 필수 검증: 매각·둘다 미입력 시 다음 차단(모두 화법).
5. 홈 헤더: occupancy 저장 생기니 공실 카운트 복원 — "상가 N개 · 공실 N · 임대 중 N · 매각 N" Code 자율(과밀이면 축약).

## 테스트: 임차 있음/공실 × 임대/매각/둘다 — 필수 차단·계산·라벨. 전체.
