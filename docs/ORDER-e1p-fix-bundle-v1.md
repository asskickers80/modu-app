## 1부: 등기부·건축물대장 카드 — "(미구현)" 표기만
- 기존 카드 유지, 손대지 말 것. 제목 뒤에 "(미구현)"만.
  "등기부등본 자동열람 완료 (미구현)" / 건축물대장 동일.

- 회귀: "(미구현)" 존재 확인.

## 2부: E1p 수정 모드 — 기존 상가 로드
1. E2L 소유자 "수정" → E1p가 해당 상가 값 채운 채 열림. E1 로딩·매핑 공유 가능하면 추출·재사용, E1p 전용 필드(deal_type·매매가·캡레이트·권장업종)만 확장. 복제 금지.
2. 저장 = UPDATE(신규 INSERT 금지). 소유자 검증 isOwnerOf(user_id 우선).
3. 진입: E2L 소유자 액션 → /e1p?edit={id}. 홈 상가 카드·가이드 딥링크 중 수정 진입 자연스러운 곳 갱신.
4. 예시 상가 수정: 양도인 동일 정책(실주소 선택 시 isDemo 해제).
