# ORDER-gate-signup-skip-v1 — 행동 게이트 가입 진입 시 온보딩 생략

## 증상 (실기기 재현)
방문자(역할 선택 완료)가 E2 문의 → 가입 유도 시트 → "가입하기" → 온보딩 처음(역할 선택)부터 재시작.

## 결정 (대표 확정)
행동 게이트에서 진입한 가입은 온보딩 전체를 건너뛰고 곧장 회원가입 화면으로.
- 이미 선택된 역할(방문자 포함) 유지. 역할 재질문 금지.
- 가입 완료 → returnTo 복귀 (?contact=1 등 의도 파라미터 유지, 기존 구현 재사용).

## 구현 지시
1. 게이트 시트 "가입하기" 라우팅: 온보딩 엔트리 대신 가입 화면 직행. 진입 출처(게이트 vs 일반)
   구분 파라미터로 처리하되, 일반 신규 진입의 온보딩 플로우는 무변경.
2. 역할 미선택 상태로 게이트에 도달하는 경로가 있는지 조사 — 있으면 그 경우만 역할 선택 1회 노출
   (전체 온보딩 아님). 없으면 생략.
3. Playwright: 방문자 역할 → E2 문의 → 가입 직행(역할 화면 미노출) → E2 복귀+문의 시트 오픈 검증.

## 추가 항목
4. 방문자 온보딩 마지막 단계에 두 경로 제공:
   - "가입 없이 둘러보기" (주 동선, 현행처럼 바로 입장)
   - "회원가입" (방문자 역할 유지한 채 가입 화면 직행 → 완료 후 첫 화면 입장)
   둘 다 칩/버튼, 모두 화법. 가입 강요 인상 금지 — 둘러보기가 시각적 기본.
5. Playwright: 방문자 온보딩에서 (a) 둘러보기 → 비로그인 입장, (b) 회원가입 → 가입 후 입장, 2케이스 추가.
