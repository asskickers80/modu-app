# SIM-STARTUP-v1 — 창업준비 축 실기기 시뮬레이션 브리프

> 대표가 실기기(사파리 직접)로 밟을 창업준비 시나리오. 양도인 세션 방식 그대로.
> 정식 주소: https://modu-app-asskickers80s-projects.vercel.app

## 시나리오 순서 & 단계별 확인 포인트

### 1. 신규 이메일 가입
- A2(구름) → **"창업자"** 선택 → 다음 → A3(창업 질문) → A4 → **이메일 회원가입**(신규 주소).
- ✅ 확인: 가입 완료 후 로그인 상태 유지(로그아웃으로 안 튕김). 마이(⋯) 최하단에 **로그아웃** 보임 = 로그인 성공.
- ✅ 확인: A4에서 역할이 **창업자로 유지**(seller로 안 바뀜).

### 2. 온보딩
- A3 3문항: **방식(직영/프랜차이즈)** · **지역** · **예산**.
- ✅ 확인: 각 단계 뒤로가기(←) 정상, 이탈 후 재진입 시 답 유지/재질문.
- ✅ 확인: 업종 입력은 없음(정상 — 창업준비는 소비축).

### 3. 홈 (추천 피드, /a7/startup)
- ✅ 확인: 상단 **"오늘의 인사이트"**(모두 심볼 뱃지) · **"창업 준비 진단"** — **"AI" 글자 없음**.
- ✅ 확인 (데모 표식): **빈 점포**·**프랜차이즈 추천** 카드는 현재 예시 데이터(아래 표). 실양도 매물만 실데이터.

### 4. 탐색 (/explore)
- 하단 **탐색** 탭 → 매물 목록.
- ✅ 확인: 협의중 매물에 **"협의 중" 뱃지**.

### 5. 매물 상세 (E2) 열람
- 아무 매물 카드 탭 → 상세.
- ✅ 확인: 로그인 상태라 **자유 열람**(게이트 없음).

### 6. 문의 발신 (핵심)
- 상세 하단 **"DM으로 문의하기"** → **"DM 대화 시작하기"**.
- ✅ 확인: **"가입이 필요해요" 게이트가 뜨지 않음**(로그인 상태). 바로 대화방(/d4/chat) 진입.

### 7. 답장 수신 확인 (반대편)
- (양도인 계정/기기에서) 메시지 탭 → 인박스.
- ✅ 확인: 방금 보낸 문의가 **양도인 인박스에 뜸**(상대="창업자" 이름 + 빨간/축색 안읽음 점).
- ✅ 확인: 스레드 진입 시 발신자 좌/우 정렬 정확(문의자=우측, 양도인 답장=좌측).

### 8. 관심/찜 동작
- 홈·탐색 카드의 하트(♥) 탭.
- ⚠️ 알려진 한계: 현재 찜은 **로컬 표시만**(서버 저장·게이트 미구현 — LOGIN-DEBT.md #7). 새로고침 시 초기화 가능. 방향 결정 대기.

## 로그인 세션 진단 참고
- 임시 진단 페이지(/dev/supabase)는 **제거됨**(라우트 사망). 로그인 여부는 **마이 ⋯의 로그아웃 노출**로 확인.
