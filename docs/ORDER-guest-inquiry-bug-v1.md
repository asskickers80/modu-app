# ORDER-guest-inquiry-bug-v1 — 문의 게이트 returnTo 오작동 + 소유자 판정 뚫림 조사

## 증상 (실기기 재현됨)
방문자가 E2(왓더버거, 예시 매물)에서 문의하기 → 가입 게이트 → 가입 완료 후
문의 시트가 아니라 "매물 수정"(E1, 1/4) 화면으로 진입. 왓더버거 데이터가 폼에 채워진 상태.
즉 신규 계정이 타인 매물의 수정 화면에 들어감.

## 조사 항목 (수정 전 원인 보고 먼저)
1. returnTo 체인 추적: E2 문의 클릭 → 게이트 시트 → 가입 플로우 → 복귀. 어디서 목적지가 E1 수정으로 바뀌는지.
   - 가입 시 역할 선택(양도자 등)이 온보딩 라우팅으로 returnTo를 덮어쓰는지 확인.
2. E1 수정 진입 가드: 현재 소유자 검증이 무엇인지(기기ID? user_id? 없음?).
   - 예시(status='example') 매물이 소유자 판정을 우회하는 특례가 있는지.
3. E2 소유자 모드 판정과 E1 진입 가드가 같은 로직을 쓰는지, 복제돼 있는지 (원칙 5: 복제 금지).

## 수정 지시 (원인 보고 후 진행)
1. returnTo: 가입 완료 시 온보딩보다 returnTo가 우선. 복귀 목적지는 E2 + 문의 시트 오픈 상태.
2. E1 수정 진입: 소유자 아니면 차단(E2로 리다이렉트). 예시 매물도 동일 — 예시라고 수정 개방 금지.
3. 소유자 판정 로직은 단일 함수로 추출해 E2 소유자 모드와 E1 가드가 공유.
4. Playwright: (a) 방문자 문의→가입→E2 문의 시트 복귀, (b) 비소유자 E1 URL 직접 진입 시 차단, 2케이스 추가.
