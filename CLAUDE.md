# 모두(Modu) — 프로젝트 가이드 (CLAUDE.md)

## 이 프로젝트가 뭔가
"모두"는 한국 자영업자(점포 기반 200만)를 위한 AI 리테일 생태계 슈퍼앱. 점포 양도/양수, 창업, 임대, 운영, 그리고 이들을 돕는 기업회원(업체)을 하나로 잇는다. 핵심 무기는 "AI 정제 양면 매칭" — 수요자와 업체를 AI가 골라 연결하는 인프라.

## 작업 방식 (중요)
- 기획·설계는 이미 끝났다(별도 인수인계 문서 v16 존재). 지금은 실제 코딩 단계.
- **A1(첫 진입 화면)부터 화면 순서대로 하나씩** 만든다. 한 화면을 만들고, 거기서 API·동작·엣지케이스·UX를 깊이 고민해 완성한 뒤 다음 화면으로.
- 한 번에 다 만들지 말 것. 한 화면씩, 천천히, 확인받으며 진행.
- 사용자(대표님)는 코딩 비전문가다. 전문 용어는 풀어서 설명하고, 무엇을 왜 하는지 알려주며 진행할 것.
- 답변은 간결하게. 호칭은 "대표님".

## 기술 스택
- React + Vite + Tailwind CSS (웹 우선, 모바일 화면 비율 390px 기준으로 디자인)
- React Router (화면 간 이동)
- 처음엔 화면·흐름부터. 백엔드·DB·공공 API는 필요해지는 화면(매물 등록 E1)에서 붙인다.
- 나중에 React Native로 앱 전환 가능성을 염두에 두고 구조를 짠다.

## 첫 진입 흐름 (A 시리즈 — 여기부터 만든다)
- A1 스플래시 (로고 확정 후, 일단 보류)
- A2 카테고리 선택 (위 6개 칩, 중복 선택 가능, 배민 톤)
- A3 카테고리별 핵심 질문 3개 (칩·탭으로 답하기, 타이핑 최소화)
- A4 가입 방식 (카카오/네이버/Apple/휴대폰)
- A6 환영 (로고 후 보류)
- A7 첫 화면 (카테고리별 대시보드 / 창업준비는 추천 피드 / 기업회원은 영업 상황판)

## 하단 네비게이션 5탭
홈 / 탐색 / 커뮤니티(오픈채팅) / 메시지(DM) / 마이

## 절대 지킬 핵심 원칙
- **DM 우선**: 모든 거래 소통은 전화번호 아닌 앱 내 DM으로 시작. 번호는 비공개가 기본.
- **연락처 교환 = 요청→수락** (양쪽 동시 공개, 일방 불가). 누르기 전 모달로 미리 고지.
- **B2C 거래엔 '매칭 성사' 선언 없음** (계약은 앱 밖 현실). 업체 매칭(수요자↔기업회원)에만 성사 개념.
- **타이핑 최소화**: 칩·버튼 탭이 1순위, 직접입력은 최후.
- **노출 3층**: 무료/프리미엄/광고. 노출 순위 직접 판매 금지.
- **입력 절감**: 주소만 넣으면 공공데이터(건축물대장 등) 자동 채움.

## UI 톤
- 토스 패턴: 제목 라벨 대신 데이터 헤드라인, 변화량(↑↓%) 표시.
- 깔끔하고 모던하게. 모바일 우선. 터치 영역 넉넉히(44px).
